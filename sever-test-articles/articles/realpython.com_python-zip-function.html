<p>Python&rsquo;s <code>zip()</code> function creates an iterator that will aggregate elements from two or more iterables. You can use the resulting iterator to quickly and consistently solve common programming problems, like creating <a href="https://realpython.com/courses/dictionaries-python/">dictionaries</a>. In this tutorial, you&rsquo;ll discover the logic behind the Python <code>zip()</code> function and how you can use it to solve real-world problems.</p>
<p><strong>By the end of this tutorial, you&rsquo;ll learn:</strong></p>
<ul>
<li>How <strong><code>zip()</code></strong> works in both Python 3 and Python 2</li>
<li>How to use the Python <code>zip()</code> function for <strong>parallel iteration</strong></li>
<li>How to <strong>create dictionaries</strong> on the fly using <code>zip()</code></li>
</ul>

<div class="w-100 text-center js-needs-scaling" style="transform-origin: 0 0;"><div id="waldo-tag-4996"></div> <a class="small text-muted js-disclosure" href="/account/join/" rel="nofollow" style="display: none;"> <i aria-hidden="true" class="fa fa-info-circle"> </i> Remove ads</a></div><h2 id="understanding-the-python-zip-function">Understanding the Python <code>zip()</code> Function</h2>
<p><code>zip()</code> is available in the <a href="https://docs.python.org/3/library/builtins.html">built-in namespace</a>. If you use <code>dir()</code> to inspect <code>__builtins__</code>, then you&rsquo;ll see <code>zip()</code> at the end of the list:</p>
<div class="highlight python repl"><span class="repl-toggle" title="Toggle REPL prompts and output">&gt;&gt;&gt;</span><pre><span></span><span class="gp">&gt;&gt;&gt; </span><span class="nb">dir</span><span class="p">(</span><span class="n">__builtins__</span><span class="p">)</span>
<span class="go">[&#39;ArithmeticError&#39;, &#39;AssertionError&#39;, &#39;AttributeError&#39;, ..., &#39;zip&#39;]</span>
</pre></div>
<p>You can see that <code>'zip'</code> is the last entry in the list of available objects.</p>
<p>According to the <a href="https://docs.python.org/3/library/functions.html#zip">official documentation</a>, Python&rsquo;s <code>zip()</code> function behaves as follows:</p>
<blockquote>
<p>Returns an iterator of tuples, where the <em>i</em>-th tuple contains the <em>i</em>-th element from each of the argument sequences or iterables. The iterator stops when the shortest input iterable is exhausted. With a single iterable argument, it returns an iterator of 1-tuples. With no arguments, it returns an empty iterator. (<a href="https://docs.python.org/3/library/functions.html#zip">Source</a>)</p>
</blockquote>
<p>You&rsquo;ll unpack this definition throughout the rest of the tutorial. As you work through the code examples, you&rsquo;ll see that Python zip operations work just like the physical zipper on a bag or pair of jeans. Interlocking pairs of teeth on both sides of the zipper are pulled together to close an opening. In fact, this visual analogy is perfect for understanding <code>zip()</code>, since the function was named after physical zippers!</p>
<h2 id="using-zip-in-python">Using <code>zip()</code> in Python</h2>
<p>Python&rsquo;s <code>zip()</code> function is defined as <code>zip(*iterables)</code>. The function takes in <a href="https://docs.python.org/3/glossary.html#term-iterable">iterables</a> as arguments and returns an <strong>iterator</strong>. This iterator generates a series of tuples containing elements from each iterable. <code>zip()</code> can accept any type of iterable, such as <a href="https://realpython.com/read-write-files-python/">files</a>, <a href="https://realpython.com/python-lists-tuples/">lists, tuples</a>, <a href="https://realpython.com/python-dicts/">dictionaries</a>, <a href="https://realpython.com/python-sets/">sets</a>, and so on.</p>
<h3 id="passing-n-arguments">Passing <code>n</code> Arguments</h3>
<p>If you use <code>zip()</code> with <code>n</code> arguments, then the function will return an iterator that generates tuples of length <code>n</code>. To see this in action, take a look at the following code block:</p>
<div class="highlight python repl"><span class="repl-toggle" title="Toggle REPL prompts and output">&gt;&gt;&gt;</span><pre><span></span><span class="gp">&gt;&gt;&gt; </span><span class="n">numbers</span> <span class="o">=</span> <span class="p">[</span><span class="mi">1</span><span class="p">,</span> <span class="mi">2</span><span class="p">,</span> <span class="mi">3</span><span class="p">]</span>
<span class="gp">&gt;&gt;&gt; </span><span class="n">letters</span> <span class="o">=</span> <span class="p">[</span><span class="s1">&#39;a&#39;</span><span class="p">,</span> <span class="s1">&#39;b&#39;</span><span class="p">,</span> <span class="s1">&#39;c&#39;</span><span class="p">]</span>
<span class="gp">&gt;&gt;&gt; </span><span class="n">zipped</span> <span class="o">=</span> <span class="nb">zip</span><span class="p">(</span><span class="n">numbers</span><span class="p">,</span> <span class="n">letters</span><span class="p">)</span>
<span class="gp">&gt;&gt;&gt; </span><span class="n">zipped</span> <span class="c1"># Holds an iterator object</span>
<span class="go">&lt;zip object at 0x7fa4831153c8&gt;</span>
<span class="gp">&gt;&gt;&gt; </span><span class="nb">type</span><span class="p">(</span><span class="n">zipped</span><span class="p">)</span>
<span class="go">&lt;class &#39;zip&#39;&gt;</span>
<span class="gp">&gt;&gt;&gt; </span><span class="nb">list</span><span class="p">(</span><span class="n">zipped</span><span class="p">)</span>
<span class="go">[(1, &#39;a&#39;), (2, &#39;b&#39;), (3, &#39;c&#39;)]</span>
</pre></div>
<p>Here, you use <code>zip(numbers, letters)</code> to create an iterator that produces tuples of the form <code>(x, y)</code>. In this case, the <code>x</code> values are taken from <code>numbers</code> and the <code>y</code> values are taken from <code>letters</code>. Notice how the Python <code>zip()</code> function returns an iterator. To retrieve the final list object, you need to use <code>list()</code> to consume the iterator.</p>
<p>If you&rsquo;re working with sequences like lists, tuples, or <a href="https://realpython.com/python-strings/">strings</a>, then your iterables are guaranteed to be evaluated from left to right. This means that the resulting list of tuples will take the form <code>[(numbers[0], letters[0]), (numbers[1], letters[1]),..., (numbers[n], letters[n])]</code>. However, for other types of iterables (like <a href="https://realpython.com/python-sets/">sets</a>), you might see some weird results:</p>
<div class="highlight python repl"><span class="repl-toggle" title="Toggle REPL prompts and output">&gt;&gt;&gt;</span><pre><span></span><span class="gp">&gt;&gt;&gt; </span><span class="n">s1</span> <span class="o">=</span> <span class="p">{</span><span class="mi">2</span><span class="p">,</span> <span class="mi">3</span><span class="p">,</span> <span class="mi">1</span><span class="p">}</span>
<span class="gp">&gt;&gt;&gt; </span><span class="n">s2</span> <span class="o">=</span> <span class="p">{</span><span class="s1">&#39;b&#39;</span><span class="p">,</span> <span class="s1">&#39;a&#39;</span><span class="p">,</span> <span class="s1">&#39;c&#39;</span><span class="p">}</span>
<span class="gp">&gt;&gt;&gt; </span><span class="nb">list</span><span class="p">(</span><span class="nb">zip</span><span class="p">(</span><span class="n">s1</span><span class="p">,</span> <span class="n">s2</span><span class="p">))</span>
<span class="go">[(1, &#39;a&#39;), (2, &#39;c&#39;), (3, &#39;b&#39;)]</span>
</pre></div>
<p>In this example, <code>s1</code> and <code>s2</code> are <code>set</code> objects, which don&rsquo;t keep their elements in any particular order. This means that the tuples returned by <code>zip()</code> will have elements that are paired up randomly. If you&rsquo;re going to use the Python <code>zip()</code> function with unordered iterables like sets, then this is something to keep in mind.</p>
<h3 id="passing-no-arguments">Passing No Arguments</h3>
<p>You can call <code>zip()</code> with no arguments as well. In this case, you&rsquo;ll simply get an empty iterator:</p>
<div class="highlight python repl"><span class="repl-toggle" title="Toggle REPL prompts and output">&gt;&gt;&gt;</span><pre><span></span><span class="gp">&gt;&gt;&gt; </span><span class="n">zipped</span> <span class="o">=</span> <span class="nb">zip</span><span class="p">()</span>
<span class="gp">&gt;&gt;&gt; </span><span class="n">zipped</span>
<span class="go">&lt;zip object at 0x7f196294a488&gt;</span>
<span class="gp">&gt;&gt;&gt; </span><span class="nb">list</span><span class="p">(</span><span class="n">zipped</span><span class="p">)</span>
<span class="go">[]</span>
</pre></div>
<p>Here, you call <code>zip()</code> with no arguments, so your <code>zipped</code> variable holds an empty iterator. If you consume the iterator with <code>list()</code>, then you&rsquo;ll see an empty list as well.</p>
<p>You could also try to force the empty iterator to yield an element directly. In this case, you&rsquo;ll get a <code>StopIteration</code> <a href="https://realpython.com/python-exceptions/">exception</a>:</p>
<div class="highlight python repl"><span class="repl-toggle" title="Toggle REPL prompts and output">&gt;&gt;&gt;</span><pre><span></span><span class="gp">&gt;&gt;&gt; </span><span class="n">zipped</span> <span class="o">=</span> <span class="nb">zip</span><span class="p">()</span>
<span class="gp">&gt;&gt;&gt; </span><span class="nb">next</span><span class="p">(</span><span class="n">zipped</span><span class="p">)</span>
<span class="gt">Traceback (most recent call last):</span>
File <span class="nb">&quot;&lt;stdin&gt;&quot;</span>, line <span class="m">1</span>, in <span class="n">&lt;module&gt;</span>
<span class="gr">StopIteration</span>
</pre></div>
<p>When you call <a href="https://docs.python.org/3/library/functions.html#next"><code>next()</code></a> on <code>zipped</code>, Python tries to retrieve the next item. However, since <code>zipped</code> holds an empty iterator, there&rsquo;s nothing to pull out, so Python raises a <code>StopIteration</code> exception.</p>
<div class="w-100 text-center js-needs-scaling" style="transform-origin: 0 0;"><div id="waldo-tag-4998"></div> <a class="small text-muted js-disclosure" href="/account/join/" rel="nofollow" style="display: none;"> <i aria-hidden="true" class="fa fa-info-circle"> </i> Remove ads</a></div><h3 id="passing-one-argument">Passing One Argument</h3>
<p>Python&rsquo;s <code>zip()</code> function can take just one argument as well. The result will be an iterator that yields a series of 1-item tuples:</p>
<div class="highlight python repl"><span class="repl-toggle" title="Toggle REPL prompts and output">&gt;&gt;&gt;</span><pre><span></span><span class="gp">&gt;&gt;&gt; </span><span class="n">a</span> <span class="o">=</span> <span class="p">[</span><span class="mi">1</span><span class="p">,</span> <span class="mi">2</span><span class="p">,</span> <span class="mi">3</span><span class="p">]</span>
<span class="gp">&gt;&gt;&gt; </span><span class="n">zipped</span> <span class="o">=</span> <span class="nb">zip</span><span class="p">(</span><span class="n">a</span><span class="p">)</span>
<span class="gp">&gt;&gt;&gt; </span><span class="nb">list</span><span class="p">(</span><span class="n">zipped</span><span class="p">)</span>
<span class="go">[(1,), (2,), (3,)]</span>
</pre></div>
<p>This may not be that useful, but it still works. Perhaps you can find some use cases for this behavior of <code>zip()</code>!</p>
<p>As you can see, you can call the Python <code>zip()</code> function with as many input iterables as you need. The length of the resulting tuples will always equal the number of iterables you pass as arguments. Here&rsquo;s an example with three iterables:</p>
<div class="highlight python repl"><span class="repl-toggle" title="Toggle REPL prompts and output">&gt;&gt;&gt;</span><pre><span></span><span class="gp">&gt;&gt;&gt; </span><span class="n">integers</span> <span class="o">=</span> <span class="p">[</span><span class="mi">1</span><span class="p">,</span> <span class="mi">2</span><span class="p">,</span> <span class="mi">3</span><span class="p">]</span>
<span class="gp">&gt;&gt;&gt; </span><span class="n">letters</span> <span class="o">=</span> <span class="p">[</span><span class="s1">&#39;a&#39;</span><span class="p">,</span> <span class="s1">&#39;b&#39;</span><span class="p">,</span> <span class="s1">&#39;c&#39;</span><span class="p">]</span>
<span class="gp">&gt;&gt;&gt; </span><span class="n">floats</span> <span class="o">=</span> <span class="p">[</span><span class="mf">4.0</span><span class="p">,</span> <span class="mf">5.0</span><span class="p">,</span> <span class="mf">6.0</span><span class="p">]</span>
<span class="gp">&gt;&gt;&gt; </span><span class="n">zipped</span> <span class="o">=</span> <span class="nb">zip</span><span class="p">(</span><span class="n">integers</span><span class="p">,</span> <span class="n">letters</span><span class="p">,</span> <span class="n">floats</span><span class="p">)</span> <span class="c1"># Three input iterables</span>
<span class="gp">&gt;&gt;&gt; </span><span class="nb">list</span><span class="p">(</span><span class="n">zipped</span><span class="p">)</span>
<span class="go">[(1, &#39;a&#39;, 4.0), (2, &#39;b&#39;, 5.0), (3, &#39;c&#39;, 6.0)]</span>
</pre></div>
<p>Here, you call the Python <code>zip()</code> function with three iterables, so the resulting tuples have three elements each.</p>
<h3 id="passing-arguments-of-unequal-length">Passing Arguments of Unequal Length</h3>
<p>When you&rsquo;re working with the Python <code>zip()</code> function, it&rsquo;s important to pay attention to the length of your iterables. It&rsquo;s possible that the iterables you pass in as arguments aren&rsquo;t the same length.</p>
<p>In these cases, the number of elements that <code>zip()</code> puts out will be equal to the length of the <em>shortest</em> iterable. The remaining elements in any longer iterables will be totally ignored by <code>zip()</code>, as you can see here:</p>
<div class="highlight python repl"><span class="repl-toggle" title="Toggle REPL prompts and output">&gt;&gt;&gt;</span><pre><span></span><span class="gp">&gt;&gt;&gt; </span><span class="nb">list</span><span class="p">(</span><span class="nb">zip</span><span class="p">(</span><span class="nb">range</span><span class="p">(</span><span class="mi">5</span><span class="p">),</span> <span class="nb">range</span><span class="p">(</span><span class="mi">100</span><span class="p">)))</span>
<span class="go">[(0, 0), (1, 1), (2, 2), (3, 3), (4, 4)]</span>
</pre></div>
<p>Since <code>5</code> is the length of the first (and shortest) <a href="https://realpython.com/python-range/"><code>range()</code></a> object, <code>zip()</code> outputs a list of five tuples. There are still 95 unmatched elements from the second <code>range()</code> object. These are all ignored by <code>zip()</code> since there are no more elements from the first <code>range()</code> object to complete the pairs.</p>
<p>If trailing or unmatched values are important to you, then you can use <a href="https://docs.python.org/3/library/itertools.html#itertools.zip_longest"><code>itertools.zip_longest()</code></a> instead of <code>zip()</code>. With this function, the missing values will be replaced with whatever you pass to the <code>fillvalue</code> argument (defaults to <code>None</code>). The iteration will continue until the longest iterable is exhausted:</p>
<div class="highlight python repl"><span class="repl-toggle" title="Toggle REPL prompts and output">&gt;&gt;&gt;</span><pre><span></span><span class="gp">&gt;&gt;&gt; </span><span class="kn">from</span> <span class="nn">itertools</span> <span class="k">import</span> <span class="n">zip_longest</span>
<span class="gp">&gt;&gt;&gt; </span><span class="n">numbers</span> <span class="o">=</span> <span class="p">[</span><span class="mi">1</span><span class="p">,</span> <span class="mi">2</span><span class="p">,</span> <span class="mi">3</span><span class="p">]</span>
<span class="gp">&gt;&gt;&gt; </span><span class="n">letters</span> <span class="o">=</span> <span class="p">[</span><span class="s1">&#39;a&#39;</span><span class="p">,</span> <span class="s1">&#39;b&#39;</span><span class="p">,</span> <span class="s1">&#39;c&#39;</span><span class="p">]</span>
<span class="gp">&gt;&gt;&gt; </span><span class="n">longest</span> <span class="o">=</span> <span class="nb">range</span><span class="p">(</span><span class="mi">5</span><span class="p">)</span>
<span class="gp">&gt;&gt;&gt; </span><span class="n">zipped</span> <span class="o">=</span> <span class="n">zip_longest</span><span class="p">(</span><span class="n">numbers</span><span class="p">,</span> <span class="n">letters</span><span class="p">,</span> <span class="n">longest</span><span class="p">,</span> <span class="n">fillvalue</span><span class="o">=</span><span class="s1">&#39;?&#39;</span><span class="p">)</span>
<span class="gp">&gt;&gt;&gt; </span><span class="nb">list</span><span class="p">(</span><span class="n">zipped</span><span class="p">)</span>
<span class="go">[(1, &#39;a&#39;, 0), (2, &#39;b&#39;, 1), (3, &#39;c&#39;, 2), (&#39;?&#39;, &#39;?&#39;, 3), (&#39;?&#39;, &#39;?&#39;, 4)]</span>
</pre></div>
<p>Here, you use <code>itertools.zip_longest()</code> to yield five tuples with elements from <code>letters</code>, <code>numbers</code>, and <code>longest</code>. The iteration only stops when <code>longest</code> is exhausted. The missing elements from <code>numbers</code> and <code>letters</code> are filled with a question mark <code>?</code>, which is what you specified with <code>fillvalue</code>.</p>
<h2 id="comparing-zip-in-python-3-and-2">Comparing <code>zip()</code> in Python 3 and 2</h2>
<p>Python&rsquo;s <code>zip()</code> function works differently in both versions of the language. In Python 2, <code>zip()</code> returns a <code>list</code> of tuples. The resulting <code>list</code> is truncated to the length of the shortest input iterable. If you call <code>zip()</code> with no arguments, then you get an empty <code>list</code> in return:</p>

<p>In Python 3, however, <code>zip()</code> returns an <strong>iterator</strong>. This object yields tuples on demand and can be traversed only once. The iteration ends with a <code>StopIteration</code> exception once the shortest input iterable is exhausted. If you supply no arguments to <code>zip()</code>, then the function returns an empty iterator:</p>
<div class="highlight python repl"><span class="repl-toggle" title="Toggle REPL prompts and output">&gt;&gt;&gt;</span><pre><span></span><span class="gp">&gt;&gt;&gt; </span><span class="c1"># Python 3</span>

This will run through the iterator and return a list of tuples.</p>
<p>If you regularly use Python 2, then note that using <code>zip()</code> with long input iterables can unintentionally consume a lot of memory. <p>If you really need to write code that behaves the same way in both Python 2 and Python 3, then you can use a trick like the following:</p>
<div class="highlight python"><pre><span></span><span class="k">try</span><span class="p">:</span>

(The <a href="https://realpython.com/lessons/pass-statement/"><code>pass</code> statement</a> here is just a placeholder.) </p>
<p>With this trick, you can safely use the Python <code>zip()</code> function throughout your code. When run, your program will automatically select and use the correct version.</p>

<p>Looping over multiple iterables is one of the most common use cases for Python&rsquo;s <code>zip()</code> function. If you need to iterate through multiple lists, tuples, or any other sequence, then it&rsquo;s likely that you&rsquo;ll fall back on <code>zip()</code>. This section will show you how to use <code>zip()</code> to iterate through multiple iterables at the same time.</p>
<h3 id="traversing-lists-in-parallel">Traversing Lists in Parallel</h3>
<p>Python&rsquo;s <code>zip()</code> function allows you to iterate in parallel over two or more iterables. Since <code>zip()</code> generates tuples, you can unpack these in the header of a <a href="https://realpython.com/courses/python-for-loop/"><code>for</code> loop</a>:</p>
<div class="highlight python repl"><span class="repl-toggle" title="Toggle REPL prompts and output">&gt;&gt;&gt;</span><pre><span></span><span class="gp">&gt;&gt;&gt; </span><span class="n">letters</span> <span class="o">=</span> <span class="p">[</span><span class="s1">&#39;a&#39;</span><span class="p">,</span> <span class="s1">&#39;b&#39;</span><span class="p">,</span> <span class="s1">&#39;c&#39;</span><span class="p">]</span>

There&rsquo;s no restriction on the number of iterables you can use with Python&rsquo;s <code>zip()</code> function.</p>

<h3 id="traversing-dictionaries-in-parallel">Traversing Dictionaries in Parallel</h3>
<p>In Python 3.6 and beyond, dictionaries are <a href="https://docs.python.org/3/whatsnew/3.6.html#whatsnew36-compactdict">ordered collections</a>, meaning they keep their elements in the same order in which they were introduced. If you take advantage of this feature, then you can use the Python <code>zip()</code> function to iterate through multiple dictionaries in a safe and coherent way:</p>
<div class="highlight python repl"><span class="repl-toggle" title="Toggle REPL prompts and output">&gt;&gt;&gt;</span><pre><span></span><span class="gp">&gt;&gt;&gt; </span><span class="n">dict_one</span> <span class="o">=</span> <span class="p">{</span><span class="s1">&#39;name&#39;</span><span class="p">:</span> <span class="s1">&#39;John&#39;</span><span class="p">,</span> <span class="s1">&#39;last_name&#39;</span><span class="p">:</span> <span class="s1">&#39;Doe&#39;</span><span class="p">,</span> <span class="s1">&#39;job&#39;</span><span class="p">:</span> <span class="s1">&#39;Python Consultant&#39;</span><span class="p">}</span>
<span class="gp">&gt;&gt;&gt; </span><span class="n">dict_two</span> <span class="o">=</span> <span class="p">{</span><span class="s1">&#39;name&#39;</span><span class="p">:</span> <span class="s1">&#39;Jane&#39;</span><span class="p">,</span> <span class="s1">&#39;last_name&#39;</span><span class="p">:</span> <span class="s1">&#39;Doe&#39;</span><span class="p">,</span> <span class="s1">&#39;job&#39;</span><span class="p">:</span> <span class="s1">&#39;Community Manager&#39;</span><span class="p">}</span>

<span class="gp">... </span> <span class="nb">print</span><span class="p">(</span><span class="n">k1</span><span class="p">,</span> <span class="s1">&#39;-&gt;&#39;</span><span class="p">,</span> <span class="n">v1</span><span class="p">)</span>

<span class="gp">...</span>
<span class="go">name -&gt; John</span>
<span class="go">name -&gt; Jane</span>
<span class="go">last_name -&gt; Doe</span>
<span class="go">last_name -&gt; Doe</span>
<span class="go">job -&gt; Python Consultant</span>
<span class="go">job -&gt; Community Manager</span>
</pre></div>
<p>Here, you iterate through <code>dict_one</code> and <code>dict_two</code> in parallel. In this case, <code>zip()</code> generates tuples with the items from both dictionaries. Then, you can unpack each tuple and gain access to the items of both dictionaries at the same time.</p>
<div class="alert alert-primary" role="alert">
<p><strong>Note:</strong> If you want to dive deeper into dictionary iteration, check out <a href="https://realpython.com/iterate-through-dictionary-python/">How to Iterate Through a Dictionary in Python</a>.</p>
</div>
<p>Notice that, in the above example, the left-to-right evaluation order is guaranteed. You can also use Python&rsquo;s <code>zip()</code> function to iterate through sets in parallel. However, you&rsquo;ll need to consider that, unlike dictionaries in Python 3.6, sets <em>don&rsquo;t</em> keep their elements in order. If you forget this detail, the final result of your program may not be quite what you want or expect.</p>
<h3 id="unzipping-a-sequence">Unzipping a Sequence</h3>
<p>There&rsquo;s a question that comes up frequently in forums for new Pythonistas: &ldquo;If there&rsquo;s a <code>zip()</code> function, then why is there no <code>unzip()</code> function that does the opposite?&rdquo;</p>
<p>The reason why there&rsquo;s no <code>unzip()</code> function in Python is because the opposite of <code>zip()</code> is&hellip; well, <code>zip()</code>. Do you recall that the Python <code>zip()</code> function works just like a real zipper? The examples so far have shown you how Python zips things closed. So, how do you unzip Python objects?</p>
<p>Say you have a list of tuples and want to separate the elements of each tuple into independent sequences. To do this, you can use <code>zip()</code> along with the <a href="https://realpython.com/python-kwargs-and-args/#unpacking-with-the-asterisk-operators">unpacking operator <code>*</code></a>, like so:</p>
<div class="highlight python repl"><span class="repl-toggle" title="Toggle REPL prompts and output">&gt;&gt;&gt;</span><pre><span></span><span class="gp">&gt;&gt;&gt; </span><span class="n">pairs</span> <span class="o">=</span> <span class="p">[(</span><span class="mi">1</span><span class="p">,</span> <span class="s1">&#39;a&#39;</span><span class="p">),</span> <span class="p">(</span><span class="mi">2</span><span class="p">,</span> <span class="s1">&#39;b&#39;</span><span class="p">),</span> <span class="p">(</span><span class="mi">3</span><span class="p">,</span> <span class="s1">&#39;c&#39;</span><span class="p">),</span> <span class="p">(</span><span class="mi">4</span><span class="p">,</span> <span class="s1">&#39;d&#39;</span><span class="p">)]</span>
<span class="gp">&gt;&gt;&gt; </span><span class="n">numbers</span><span class="p">,</span> <span class="n">letters</span> <span class="o">=</span> <span class="nb">zip</span><span class="p">(</span><span class="o">*</span><span class="n">pairs</span><span class="p">)</span>
<span class="gp">&gt;&gt;&gt; </span><span class="n">numbers</span>
<span class="go">(1, 2, 3, 4)</span>
<span class="gp">&gt;&gt;&gt; </span><span class="n">letters</span>
<span class="go">(&#39;a&#39;, &#39;b&#39;, &#39;c&#39;, &#39;d&#39;)</span>
</pre></div>
<p>Here, you have a <code>list</code> of tuples containing some kind of mixed data. Then, you use the unpacking operator <code>*</code> to unzip the data, creating two different lists (<code>numbers</code> and <code>letters</code>).</p>
<div class="w-100 text-center js-needs-scaling" style="transform-origin: 0 0;"><div id="waldo-tag-5002"></div> <a class="small text-muted js-disclosure" href="/account/join/" rel="nofollow" style="display: none;"> <i aria-hidden="true" class="fa fa-info-circle"> </i> Remove ads</a></div><h3 id="sorting-in-parallel">Sorting in Parallel</h3>
<p><a href="https://realpython.com/python-sort/">Sorting</a> is a common operation in programming. Suppose you want to combine two lists and sort them at the same time. To do this, you can use <code>zip()</code> along with <code>.sort()</code> as follows:</p>
<div class="highlight python repl"><span class="repl-toggle" title="Toggle REPL prompts and output">&gt;&gt;&gt;</span><pre><span></span><span class="gp">&gt;&gt;&gt; </span><span class="n">letters</span> <span class="o">=</span> <span class="p">[</span><span class="s1">&#39;b&#39;</span><span class="p">,</span> <span class="s1">&#39;a&#39;</span><span class="p">,</span> <span class="s1">&#39;d&#39;</span><span class="p">,</span> <span class="s1">&#39;c&#39;</span><span class="p">]</span>
<span class="gp">&gt;&gt;&gt; </span><span class="n">numbers</span> <span class="o">=</span> <span class="p">[</span><span class="mi">2</span><span class="p">,</span> <span class="mi">4</span><span class="p">,</span> <span class="mi">3</span><span class="p">,</span> <span class="mi">1</span><span class="p">]</span>
<span class="gp">&gt;&gt;&gt; </span><span class="n">data1</span> <span class="o">=</span> <span class="nb">list</span><span class="p">(</span><span class="nb">zip</span><span class="p">(</span><span class="n">letters</span><span class="p">,</span> <span class="n">numbers</span><span class="p">))</span>
<span class="gp">&gt;&gt;&gt; </span><span class="n">data1</span>
<span class="go">[(&#39;b&#39;, 2), (&#39;a&#39;, 4), (&#39;d&#39;, 3), (&#39;c&#39;, 1)]</span>
<span class="gp">&gt;&gt;&gt; </span><span class="n">data1</span><span class="o">.</span><span class="n">sort</span><span class="p">()</span> <span class="c1"># Sort by letters</span>
<span class="gp">&gt;&gt;&gt; </span><span class="n">data1</span>
<span class="go">[(&#39;a&#39;, 4), (&#39;b&#39;, 2), (&#39;c&#39;, 1), (&#39;d&#39;, 3)]</span>
<span class="gp">&gt;&gt;&gt; </span><span class="n">data2</span> <span class="o">=</span> <span class="nb">list</span><span class="p">(</span><span class="nb">zip</span><span class="p">(</span><span class="n">numbers</span><span class="p">,</span> <span class="n">letters</span><span class="p">))</span>
<span class="gp">&gt;&gt;&gt; </span><span class="n">data2</span>
<span class="go">[(2, &#39;b&#39;), (4, &#39;a&#39;), (3, &#39;d&#39;), (1, &#39;c&#39;)]</span>
<span class="gp">&gt;&gt;&gt; </span><span class="n">data2</span><span class="o">.</span><span class="n">sort</span><span class="p">()</span> <span class="c1"># Sort by numbers</span>
<span class="gp">&gt;&gt;&gt; </span><span class="n">data2</span>
<span class="go">[(1, &#39;c&#39;), (2, &#39;b&#39;), (3, &#39;d&#39;), (4, &#39;a&#39;)]</span>
</pre></div>
<p>In this example, you first combine two lists with <code>zip()</code> and sort them. Notice how <code>data1</code> is sorted by <code>letters</code> and <code>data2</code> is sorted by <code>numbers</code>.</p>
<p>You can also use <code>sorted()</code> and <code>zip()</code> together to achieve a similar result:</p>
<div class="highlight python repl"><span class="repl-toggle" title="Toggle REPL prompts and output">&gt;&gt;&gt;</span><pre><span></span><span class="gp">&gt;&gt;&gt; </span><span class="n">letters</span> <span class="o">=</span> <span class="p">[</span><span class="s1">&#39;b&#39;</span><span class="p">,</span> <span class="s1">&#39;a&#39;</span><span class="p">,</span> <span class="s1">&#39;d&#39;</span><span class="p">,</span> <span class="s1">&#39;c&#39;</span><span class="p">]</span>
<span class="gp">&gt;&gt;&gt; </span><span class="n">numbers</span> <span class="o">=</span> <span class="p">[</span><span class="mi">2</span><span class="p">,</span> <span class="mi">4</span><span class="p">,</span> <span class="mi">3</span><span class="p">,</span> <span class="mi">1</span><span class="p">]</span>
<span class="gp">&gt;&gt;&gt; </span><span class="n">data</span> <span class="o">=</span> <span class="nb">sorted</span><span class="p">(</span><span class="nb">zip</span><span class="p">(</span><span class="n">letters</span><span class="p">,</span> <span class="n">numbers</span><span class="p">))</span> <span class="c1"># Sort by letters</span>
<span class="gp">&gt;&gt;&gt; </span><span class="n">data</span>
<span class="go">[(&#39;a&#39;, 4), (&#39;b&#39;, 2), (&#39;c&#39;, 1), (&#39;d&#39;, 3)]</span>
</pre></div>
<p>In this case, <code>sorted()</code> runs through the iterator generated by <code>zip()</code> and sorts the items by <code>letters</code>, all in one go. This approach can be a little bit faster since you&rsquo;ll need only two function calls: <code>zip()</code> and <code>sorted()</code>.</p>
<p>With <code>sorted()</code>, you&rsquo;re also writing a more general piece of code. This will allow you to sort any kind of sequence, not just lists.</p>

<h3 id="building-dictionaries">Building Dictionaries</h3>
<p>Python&rsquo;s <a href="https://realpython.com/python-dicts/">dictionaries</a> are a very useful data structure. Sometimes, you might need to build a dictionary from two different but closely related sequences. A convenient way to achieve this is to use <code>dict()</code> and <code>zip()</code> together. For example, suppose you retrieved a person&rsquo;s data from a form or a database. Now you have the following lists of data:</p>
<div class="highlight python repl"><span class="repl-toggle" title="Toggle REPL prompts and output">&gt;&gt;&gt;</span><pre><span></span><span class="gp">&gt;&gt;&gt; </span><span class="n">fields</span> <span class="o">=</span> <span class="p">[</span><span class="s1">&#39;name&#39;</span><span class="p">,</span> <span class="s1">&#39;last_name&#39;</span><span class="p">,</span> <span class="s1">&#39;age&#39;</span><span class="p">,</span> <span class="s1">&#39;job&#39;</span><span class="p">]</span>
<span class="gp">&gt;&gt;&gt; </span><span class="n">values</span> <span class="o">=</span> <span class="p">[</span><span class="s1">&#39;John&#39;</span><span class="p">,</span> <span class="s1">&#39;Doe&#39;</span><span class="p">,</span> <span class="s1">&#39;45&#39;</span><span class="p">,</span> <span class="s1">&#39;Python Developer&#39;</span><span class="p">]</span>
</pre></div>
<p>With this data, you need to create a dictionary for further processing. In this case, you can use <code>dict()</code> along with <code>zip()</code> as follows:</p>
<div class="highlight python repl"><span class="repl-toggle" title="Toggle REPL prompts and output">&gt;&gt;&gt;</span><pre><span></span><span class="gp">&gt;&gt;&gt; </span><span class="n">a_dict</span> <span class="o">=</span> <span class="nb">dict</span><span class="p">(</span><span class="nb">zip</span><span class="p">(</span><span class="n">fields</span><span class="p">,</span> <span class="n">values</span><span class="p">))</span>
<span class="gp">&gt;&gt;&gt; </span><span class="n">a_dict</span>
<span class="go">{&#39;name&#39;: &#39;John&#39;, &#39;last_name&#39;: &#39;Doe&#39;, &#39;age&#39;: &#39;45&#39;, &#39;job&#39;: &#39;Python Developer&#39;}</span>
</pre></div>
<p>Here, you create a dictionary that combines the two lists. <code>zip(fields, values)</code> returns an iterator that generates 2-items tuples. If you call <code>dict()</code> on that iterator, then you&rsquo;ll be building the dictionary you need. The elements of <code>fields</code> become the dictionary&rsquo;s keys, and the elements of <code>values</code> represent the values in the dictionary.</p>
<p>You can also update an existing dictionary by combining <code>zip()</code> with <code>dict.update()</code>. Suppose that John changes his job and you need to update the dictionary. You can do something like the following:</p>
<div class="highlight python repl"><span class="repl-toggle" title="Toggle REPL prompts and output">&gt;&gt;&gt;</span><pre><span></span><span class="gp">&gt;&gt;&gt; </span><span class="n">new_job</span> <span class="o">=</span> <span class="p">[</span><span class="s1">&#39;Python Consultant&#39;</span><span class="p">]</span>
<span class="gp">&gt;&gt;&gt; </span><span class="n">field</span> <span class="o">=</span> <span class="p">[</span><span class="s1">&#39;job&#39;</span><span class="p">]</span>
<span class="gp">&gt;&gt;&gt; </span><span class="n">a_dict</span><span class="o">.</span><span class="n">update</span><span class="p">(</span><span class="nb">zip</span><span class="p">(</span><span class="n">field</span><span class="p">,</span> <span class="n">new_job</span><span class="p">))</span>
<span class="gp">&gt;&gt;&gt; </span><span class="n">a_dict</span>
<span class="go">{&#39;name&#39;: &#39;John&#39;, &#39;last_name&#39;: &#39;Doe&#39;, &#39;age&#39;: &#39;45&#39;, &#39;job&#39;: &#39;Python Consultant&#39;}</span>
</pre></div>
<p>Here, <code>dict.update()</code> updates the dictionary with the key-value tuple you created using Python&rsquo;s <code>zip()</code> function. With this technique, you can easily overwrite the value of <code>job</code>.</p>
